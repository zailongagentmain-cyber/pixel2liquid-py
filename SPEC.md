# Pixel2Liquid-Py - 网站核心资源采集工具

## 项目概述

**核心目标**：准确获取网站的核心资源（HTML/CSS/JS/Images/Fonts）

**项目定位**：一个专注于资源采集的 CLI 工具，不做元素库管理，不做可视化后台（这两者独立成新项目）

---

## Phase 定义（核心目标）

### Phase 1: 网页采集
**目标**：完整采集目标网站的页面和资源

| 功能 | 说明 | 验证标准 |
|------|------|----------|
| 页面采集 | HTML + 内联资源 | 成功率 >95% |
| 资源采集 | CSS/JS/Images/Fonts | 下载完整率 >95% |
| 链接追踪 | 站内链接跟随 | 不漏页面 |
| 进度追踪 | 支持断点续传 | 中断后可恢复 |

**交付物**：本地完整镜像（HTML + 资源文件）

---

### Phase 2: 格式转换
**目标**：将采集的 HTML 转换为可用的目标格式

| 功能 | 说明 | 验证标准 |
|------|------|----------|
| HTML → Liquid | 模板语法转换 | 转换正确率 >90% |
| 路径修复 | 相对路径 → 绝对路径 | 链接不出错 |
| CDN URL 替换 | 原始 CDN → 本地/目标 CDN | 替换正确率 >95% |

**交付物**：转换后的模板文件

---

### Phase 3: CDN 处理
**目标**：管理和替换资源引用

| 功能 | 说明 | 验证标准 |
|------|------|----------|
| 资源去重 | 相同资源不重复下载 | 去重率 >80% |
| URL 映射 | original → normalized → local | 映射表完整 |
| 协议修复 | //cdn.shopify.com → https:// | 修复正确 |

**交付物**：URL 映射表 + 去重后的资源

---

### Phase 4: Shopify 适配
**目标**：生成可导入 Shopify 的主题结构

| 功能 | 说明 | 验证标准 |
|------|------|----------|
| 主题结构 | 标准 Shopify 目录结构 | 结构正确 |
| 资产文件 | CSS/JS 上传到 Shopify CDN | 可访问 |
| CLI 推送 | 通过 Shopify CLI 上传 | 推送成功 |

**交付物**：可直接导入的 .zip 或 CLI 推送成功

---

### Phase 5: CLI 工具化
**目标**：提供完整的命令行工具

| 功能 | 说明 | 验证标准 |
|------|------|----------|
| 命令行接口 | 清晰的 CLI 命令 | 可用性良好 |
| 配置文件 | 支持配置文件 | 配置生效 |
| 帮助文档 | --help 完整 | 可自助使用 |

**交付物**：可分发的 CLI 工具

---

| Phase | 指标 | 标准 |
|-------|------|------|
| Phase 1 | 网页采集成功率 | >95% |
| Phase 2 | 转换完整度 | >90% |

## 技术栈

| 层级 | 技术选择 |
|------|----------|
| 语言 | Python 3.11+ |
| 包管理 | uv |
| HTTP 客户端 | httpx / aiohttp |
| HTML 解析 | BeautifulSoup / lxml |
| CSS 处理 | cssutils |
| CLI 框架 | Click / Typer |
| 测试 | pytest |
